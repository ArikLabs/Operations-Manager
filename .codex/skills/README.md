# Local Skills

Use this directory only for repository-local skills that add real reusable leverage.

Rules:
- keep product, architecture, and delivery truth in the canonical docs
- create a local skill only when normal markdown docs are not enough
- give each skill its own folder with a `SKILL.md`
- keep skills narrow, repo-specific, and easy to delete when they go stale
- during the planning-first phase, prefer guidance and references over executable scripts unless a task explicitly requires more

Default path for a new skill:
- `.codex/skills/<skill-name>/SKILL.md`
