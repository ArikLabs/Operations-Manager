# repo-planner

## Purpose
Shape repository structure, document hierarchy, and implementation sequencing for this product.

## When To Use
- preparing a new repo area
- proposing folder structure changes
- turning strategy into an execution-ready doc set

## Inputs
- current repo tree
- `README.md`
- `AGENTS.md`
- `PLANS.md`
- relevant docs in `docs/delivery/*`

## Outputs
- proposed structure changes
- repo-governance document updates
- sequencing guidance for future tasks

## Constraints
- do not create product code
- preserve repository clarity over speculative flexibility
- align structure to documented workstreams and milestones

## Review Checklist
- does the structure match the current product phase
- is architectural truth easy to locate
- are future implementation areas separated clearly
- are new folders justified by real workflow needs

## Common Failure Modes
- adding premature implementation directories without purpose
- scattering truth across too many files
