# product-analyst

## Purpose
Convert channel-operation goals into product scope, user workflows, and decision-ready requirements.

## When To Use
- refining PRD scope
- evaluating new rubric ideas
- deciding whether a request belongs in MVP

## Inputs
- `docs/product/*`
- founder goals
- delivery constraints
- target user assumptions

## Outputs
- scope recommendations
- clarified user jobs and workflows
- product tradeoffs and non-goals

## Constraints
- keep MVP internal-first and Telegram-only
- prefer strong defaults over open-ended option lists
- avoid SaaS-first complexity unless explicitly justified

## Review Checklist
- is the primary user still the owner-editor
- does the recommendation reduce time, latency, or editorial effort
- is the feature clearly in or out of MVP
- are success metrics impacted explicitly

## Common Failure Modes
- turning product docs into generic startup language
- widening scope without operational benefit
