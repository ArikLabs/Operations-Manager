# Local Agent Team

This directory defines repo-local agent roles for coordinated work in this repository.

Rules:
- treat these files as operational tooling, not canonical truth
- read `README.md`, `AGENTS.md`, `PLANS.md`, `docs/delivery/decision-log.md`, `docs/specification.md`, and `docs/delivery/execution.md` before major work
- keep durable product, architecture, workflow, and repo decisions in the canonical docs
- use the project manager to shape scope, sequencing, dependencies, acceptance, and handoffs
- use the backend and frontend roles only when implementation is explicitly authorized
- use the tester to review risk, validation, and regressions before work is considered done
- attach local skills later only when they materially improve one of these roles

- `frontend-developer.md`
